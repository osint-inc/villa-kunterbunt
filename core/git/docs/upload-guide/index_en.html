Process flow of document submission

1Ô∏è‚É£ File Preparation

Before uploading a file, ensure it complies with the required naming scheme. Every new document must be prefixed with NEW_. This prefix serves as an entry marker for the administration.

Filename: NEW_Filename.Extension

Examples:

NEW_FinalReport_CompanyXY.pdf
NEW_Documentation.docx
NEW_Instructions.md

2Ô∏è‚É£ Target Directory

All documents must be placed in the following path within the repository:

core/git/reports/

3Ô∏è‚É£ Submission Process (Step by Step)

  1. Access Repository:
    Navigate to the directory core/git/reports/ on GitHub.
  2. Upload File:
    • Click the "Add file" button at the top right and select "Upload files".
    • Drag the prepared file (with the NEW_ prefix) into the upload field or select it from your local computer.
  3. Create Commit:
    • Enter a brief, precise description in the "Commit changes" field (e.g., "Submission of Final Report Company XY").
    • Select the option "Create a new branch for this commit and start a pull request".
    • Click on "Propose changes".
  4. Open Pull Request (PR):
    Finally, click on "Create pull request".
